Fish Annotator — Help

What it is for

Fish Annotator lets you draw and adjust outlines (“masks”) of anatomical features on zebrafish images — body contour, eye, notochord, and so on. It saves them as a FishInspector __SHAPES.json file for analysis in FishInspector. Use it two ways: annotate an image from scratch, or load an existing mask and correct it.

1 · Prepare images: TIFF → JPG

Microscopes usually save TIFF. Fish Annotator runs fastest on JPG, so convert first:

  • Save as JPG at 90% quality or higher (the converter uses 95).
  • Keep the exact same pixel size as the original — essential, because the annotation coordinates must line up with the original for FishInspector.
  • Keep your original TIFFs; the JPG is only for annotating.

Conversion runs on a computer — see Converting TIFF → JPG below.

2 · Keep each image and its JSON together

Put each image and its annotation file <image name>__SHAPES.json in the same folder. When you open them together, Fish Annotator pairs them automatically by name and shows the existing outlines. The matching is tolerant of spaces, underscores, and .ome/.tif differences.

3 · Open & save — by device

Open with Load images + JSON (select the image and its JSON in one dialog) or Load folder (loads every image + JSON in a folder, where supported). Save active JSON writes <image>__SHAPES.json.

On most devices the saved file lands in Downloads — the app cannot write back into a cloud folder — so you then move it into your image folder. (Exception: Android Chrome / desktop can save into a chosen folder.)

You open images from there; saved JSONs return through Downloads and you copy them back into the folder.

  • Flip ⇅ (first on the toolbar) — mirrors the image top-to-bottom for correct orientation, and saves it. Only available before annotating an image that has no JSON yet (greyed out once any feature is drawn or a JSON is loaded, since flipping would misalign outlines). Where the browser allows it (Android/desktop Chrome, ChromeOS, usually via Load folder) it overwrites the original file in place; otherwise it downloads the flipped JPG for you to replace the original. You then annotate the flipped image normally.
  • Draw — trace an outline with the pen, in as many strokes as you like. Each stroke is a piece; start a new stroke near the loose end of the last one and they weld together (overshoot trimmed, joint smoothed). Reposition the image between strokes and keep going all the way around; when the last piece meets the first, the loop closes and the dot turns green.
  • Edit — push an existing outline into place. A circle (the Brush) follows the pen and shoves the line out of the circle, away from the brush — like a disc pushing a string. Push from one side to move the line that way; approach from the other side to push it back. On a computer the circle follows the mouse and pushes only while a button is held; with a pen it acts only while touching.
  • Erase — a brush that cuts the active outline where you touch it, opening a real gap so you can redraw that part. Erasing can split the line into pieces; the feature dot turns amber until it is one closed loop again.
  • Join loose ends (second toolbar row, during Draw or Erase) — connects any remaining open ends of the active feature by nearest distance, closing the contour. Use it when a gap is a few pixels too wide to join on its own.
  • Clear — remove the whole active feature’s outline at once.
  • Move & zoom — on a touchscreen, two fingers move the image and pinch zooms (one finger / the pen always draws). On a computer, hold Shift and drag to move the image, use the mouse wheel or the −/100%/+/Fit buttons to zoom.
  • Second toolbar row — appears while Draw, Edit, or Erase is selected. It always shows Image view: Brightness / Contrast (with reset) — these change only how the image looks on screen, never the file. Draw and Erase also show Join loose ends; Edit and Erase show the Brush size.
  • All features — shows every outline at once for review (editing paused). Tap a feature to resume editing it.
  • Feature tiles — choose the active feature. The dot is red when empty, amber when the contour is open (has a gap), and green when it is one closed loop. Small gaps are auto-joined when you switch feature, open All features, or save; a real gap that remains blocks saving until you connect it.
  • Undo and Save active JSON. A bright Unsaved JSON badge shows when you have changes not yet saved — save after finishing each feature. Your in-progress work (including half-drawn outlines) is kept in the browser, so re-opening the same image later resumes where you left off; but only Save active JSON produces the file for FishInspector. Switching to another image with unsaved changes asks whether to save first.

    • Draw, Edit, and Erase act only on the image itself — strokes in the dark margin around it are ignored, and a line can run along the image edge but never outside it.

    5 · Typical workflow

    Load the image (and its JSON if it exists) → pick a feature → trace it, or use Edit to adjust → check every dot is green → Save active JSON → move the JSON into the folder with the original TIFF → open in FishInspector for analysis.

    Converting TIFF → JPG (on a computer)

    Do this on a PC/Mac, not the tablet. It keeps the JPG’s pixel dimensions identical to the TIFF (required for FishInspector). The drawing tool itself needs no installation — just open index.html in a browser. There is no in-page converter yet; use the convert_to_jpg.py script:

    1. Install Python 3 from python.org. On Windows, tick “Add Python to PATH” during setup.
    2. Open a terminal / Command Prompt and install the libraries:
      pip install pillow tifffile numpy
    3. Run the converter:
      python convert_to_jpg.py INPUT_FOLDER OUTPUT_FOLDER

    Defaults: JPG quality 95, grayscale, same pixel size. That is the entire setup.
